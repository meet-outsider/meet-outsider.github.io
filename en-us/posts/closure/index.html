closure

function advance: closure

May 23, 2023

  • rust
  • golang
  • js
  • ts

特性

  • 来自百度百科的定义

    闭包就是能够读取其他函数内部变量的函数。例如在 javascript 中，只有函数内部的子函数才能读取局部变量，所以闭包可以理解成“定义在一个函数内部的函数“。在本质上，闭包是将函数内部和函数外部连接起来的桥梁。

  • wiki 上的定义

    在计算机科学中，闭包（英語：Closure），又稱词法闭包（Lexical Closure）或函數閉包（function closures），是在支持头等函数的编程语言中实现词法绑定的一种技术。闭包在实现上是一个结构体，它存储了一个函数（通常是其入口地址）和一个关联的环境（相当于一个符号查找表）。环境里是若干对符号和值的对应关系，它既要包括约束变量（该函数内部绑定的符号），也要包括自由变量（在函数外部定义但在函数内被引用），有些函数也可能没有自由变量。闭包跟函数最大的不同在于，当捕捉闭包的时候，它的自由变量会在捕捉时被确定，这样即便脱离了捕捉时的上下文，它也能照常运行。捕捉时对于值的处理可以是值拷贝，也可以是名称引用，这通常由语言设计者决定，也可能由用户自行指定（如 C++）。

    可以看到，闭包的定义有两个要点：

  1. 闭包是一个结构体，它存储了一个函数（通常是其入口地址）和一个关联的环境（相当于一个符号查找表）。
  2. 闭包的自由变量会在捕捉时被确定，这样即便脱离了捕捉时的上下文，它也能照常运行。

这样做的好处是，闭包可以在不同的上下文中运行，而不会受到上下文的影响。

代码实现

接下来直接看一些例子 😊:

经典的计数器例子：

function counter() {
  let count = 0;
  return function () {
    count++;
    console.log(count);
  };
}

运行结果:

const c = counter();
c(); // 1
c(); // 2
c(); // 3
const c2 = counter();
c2(); // 1
c2(); // 2
c2(); // 3

可以看到两次调用 counter() 生成了两个不同的计数器，它们之间的状态是独立的。

步骤拆分

如果你根据上面的定义，和上面的例子，已经理解了，你很聪明 👍

如果还是不太理解或者对文章感兴趣，可以看下面的步骤拆分

这里使用 go 语言来实现上面的例子的拆分

package test

import (
	"testing"

	"github.com/go-playground/assert/v2"
)

func TestClosure(t *testing.T) {
    // 1.counter是一个函数，它返回一个函数
	Counter := func() func() int {
		freeVariable := 0
		incr := func() int {
			freeVariable++
			return freeVariable
		}
        // 返回的是incr，不是incr()
		return incr
	}

    // 2. counter1 是一个变量，它的类型是一个函数
    // 我们拆解一下
    // counter1 := Counter()
    // 因为 Counter()=调用Counter 调用Counter=incr
    // 所以 counter1 := incr
	counter1 := Counter()   // 此时 counter=ince freeVariable = 0
    // 3.接下来调用 counter1
	counter1()              // 相当于 incr() 此时  freeVariable = 1
	counter1()              // 相当于 incr() 此时  freeVariable = 2
	count := counter1()     // 相当于 incr() 此时  freeVariable = 3

    // ...counter2同上
    // ...
    //...
	assert.Equal(t, 3, count)
}

rust 中的闭包

rust闭包的定义

看看邪教的闭包写法😈

rust的闭包参数和返回值都是类型，而不是变量名

参数使用 |arg1, arg2, …| 来定义，而不是 (arg1, arg2, …)

这里先讲一下rust的闭包规则

  1. 闭包的参数和返回值都是类型，而不是变量名
  2. 闭包的参数和返回值类型必须声明，不能使用类型推导
  3. 闭包的参数使用 |arg1, arg2, …| 来定义，而不是 (arg1, arg2, …)
  4. 闭包的返回值使用 -> 来定义，而不是 :

rust闭包的triat规则

  1. FnOnce: 闭包可以被调用一次，闭包内部捕获的变量只能使用一次
  2. FnMut: 闭包可以被调用一次或多次，闭包内部捕获的变量可以使用多次
  3. Fn: 闭包可以被调用一次或多次，闭包内部捕获的变量可以使用多次，但是不能修改

可以看出Fn对应的是不可变引用，FnMut对应的是可变引用，FnOnce对应的是所有权

也就是如果你需要修改闭包内部捕获的变量，那么你就需要使用FnMut，

如果你需要获取闭包内部捕获的变量的所有权，那么你就需要使用FnOnce

如果只想使用闭包内部捕获的变量，那么你就需要使用Fn

fn main() {
    // 使用 fn
    let add_fn: fn(i32, i32) -> i32 = add;
    let result = add_fn(2, 3);
    println!("Result (fn): {}", result);

    // 使用 FnOnce
    let add_fn_once: Box<dyn FnOnce(i32, i32) -> i32> = Box::new(|a, b| a + b);
    let result_once = add_fn_once(2, 3);
    println!("Result (FnOnce): {}", result_once);
    // add_fn_once(2, 3); // 第二次调用，会报错，因为 FnOnce 只能调用一次
    // 使用 FnMut
    let mut counter = 0;
    let mut increment_fn_mut = || {
        counter += 1;
        println!("Counter: {}", counter);
    };

    increment_fn_mut(); // 第一次调用
    increment_fn_mut(); // 第二次调用

    // 使用闭包作为参数
    execute_closure(|a, b| a * b, 2, 3);
}

// 定义一个普通的函数
fn add(a: i32, b: i32) -> i32 {
    a + b
}

// 接收一个 FnOnce 闭包作为参数，并进行一次调用
fn execute_closure<F>(closure: F, a: i32, b: i32) -> i32
where
    F: FnOnce(i32, i32) -> i32,
{
    closure(a, b)
}
